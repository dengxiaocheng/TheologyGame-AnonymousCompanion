# 移动端适配计划

## 约束
- **必须纯手机浏览器可玩，禁止依赖电脑键盘（数字键、空格、回车、H 键）**
- 所有交互必须通过触摸屏完成
- 界面布局需适配竖屏手机（375px-430px 宽度）

## 当前问题
- 对话选项用数字键 1-9 选择，手机无法使用
- Space/Enter 跳过对话打字效果，手机无键盘
- H 键切换对话历史，手机无法使用
- 没有任何触摸事件
- 主文件：`js/dialogue-renderer.js`、`js/choice-system.js`、`js/ui.js`

## 修改方案

### 1. 对话选项改为可点击按钮
- `js/choice-system.js`：每个选项渲染为独立的可点击按钮
- 按钮样式：全宽，最小高度 48px，圆角，有编号和文字
- 点击即选中，不再需要数字键

### 2. 触摸跳过打字效果
- `js/dialogue-renderer.js`：点击对话文本区域 = 跳过/加速打字
- 与当前 Space/Enter 功能等效
- 添加点击涟漪效果作为视觉反馈

### 3. 对话历史改为 UI 按钮
- `js/ui.js`：在对话框右上角添加一个「历史」按钮图标
- 点击展开历史面板，再点关闭
- 不再需要 H 键

### 4. 移动端布局
- 对话框宽度 95vw，底部留安全距离（safe-area-inset）
- 文字大小 16px+（防止 iOS 自动缩放）
- 选项按钮之间间距 12px，防止误触
- `touch-action: manipulation`

### 5. 保留键盘兼容
- 桌面端仍可使用键盘快捷键
- 移动端隐藏键盘提示文字
